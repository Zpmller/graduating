# 矿业动火作业行为安全监测系统使用说明

本文档面向系统使用和演示，说明如何启动服务、配置设备、运行 Edge、查看视频和处理告警。

## 1. 系统组成

| 组件 | 作用 | 默认地址 |
| --- | --- | --- |
| 后端 Backend | API、数据库、文件、流控制 | `http://localhost:8000` |
| 前端 Dashboard | Web 管理看板 | `http://localhost:5173` |
| AI Edge | 本地视频分析、告警上传、推流 | PyQt 桌面窗口 |
| SRS | RTMP 接入和 WHEP/WebRTC 播放 | `http://localhost:1985` |

## 2. 一键启动

在项目根目录双击或执行：

```bat
start_system.bat
```

脚本会启动：

1. SRS 流媒体服务。
2. 后端 API。
3. AI Edge 桌面程序。
4. 前端开发服务。

只启动后端和 Edge：

```bat
start_ai_backend.bat
```

## 3. 首次使用流程

1. 启动 MySQL，并确认后端 `.env` 的 `DATABASE_URL` 正确。
2. 启动后端并初始化数据库。
3. 启动前端，登录系统。
4. 在设备管理页创建设备：
   - `name`：设备名称。
   - `location`：部署位置。
   - `ip_address`：摄像头或视频源地址。
   - `edge_host`：Edge 主机地址；同机演示使用 `127.0.0.1`。
5. 启动 Edge。
6. Edge bootstrap 成功后，设备会通过心跳变为在线。
7. 在前端设备页打开视频。
8. 检测到风险后，在告警历史和总览页查看告警。

## 4. 设备配置说明

`ip_address` 表示视频源地址，不一定是纯 IP，可填写 RTSP URL。

`edge_host` 表示 Edge 程序所在机器的地址，用于后端通知 Edge 推流。常见填写方式：

| 部署方式 | `edge_host` |
| --- | --- |
| 后端和 Edge 同机 | `127.0.0.1` |
| 局域网两台机器 | Edge 机器的局域网 IP |
| 固定主机名 | 后端可解析的主机名 |

## 5. Edge 使用

Edge 启动后会自动：

1. 请求后端 bootstrap。
2. 获取设备 ID、token、视频源地址。
3. 加载检测和识别模块。
4. 开始发送心跳。
5. 检测到风险时上传告警和证据图。
6. 收到前端播放请求后向 SRS 推流。

如果 bootstrap 失败，请检查：

- 后端是否启动。
- `BACKEND_URL` 是否正确。
- 前端设备中的 `edge_host` 是否与 Edge 主机匹配。
- 是否设置了 `BOOTSTRAP_SECRET` 且两端一致。

## 6. Web 看板使用

### 登录

进入 `http://localhost:5173`，使用后端初始化的账号登录。登录后 token 会用于后续 API 请求。

### 总览

查看设备状态、告警数量、最新风险和任务概况。

### 设备管理

可进行：

- 创建设备。
- 修改设备名称、位置、视频源和 Edge 主机。
- 查看设备在线状态。
- 上传标定 YAML。
- 上传标定图片。
- 打开或停止实时视频。

### 告警历史

可按设备、类型、等级、确认状态和时间范围筛选告警。告警确认后会记录确认状态和时间。

### 任务管理

可创建、更新、删除整改任务，并按状态或负责人筛选。

### 用户管理

管理员可维护用户账号、角色和启用状态。

## 7. 视频播放

播放视频前确认：

- SRS 已启动。
- Edge 在线。
- 设备视频源可读。
- FFmpeg 可用。
- 前端能访问 SRS WHEP 地址。

前端点击播放后，系统会自动完成：

```text
前端请求播放 -> 后端控制 Edge -> Edge 推 RTMP -> SRS 转 WebRTC -> 前端播放
```

## 8. 告警处理

告警产生后：

1. Edge 上传告警类型、等级、消息、时间和证据图。
2. 后端保存记录和图片。
3. 前端展示告警列表和统计。
4. 操作员确认告警。
5. 根据需要创建整改任务。

## 9. 关闭系统

- 前端开发服务：关闭对应终端窗口。
- 后端：关闭 Uvicorn 终端窗口。
- Edge：关闭 PyQt 窗口。
- SRS：

```bash
docker compose down
```

## 10. 常见故障

| 现象 | 处理 |
| --- | --- |
| 前端无法登录 | 检查后端是否运行、CORS、账号初始化 |
| 设备一直离线 | 检查 Edge bootstrap、`edge_host`、心跳日志 |
| 视频无法显示 | 检查 SRS、FFmpeg、WebRTC candidate、流状态接口 |
| 告警没有图片 | 检查后端 `static/evidence` 目录和上传响应 |
| 人脸识别无结果 | 检查人脸库图片是否上传、Edge 是否同步或可访问图片 |
