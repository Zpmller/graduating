# 3 关键功能模块设计与实现

本章在第 2 章系统设计的基础上，阐述多目标检测与气瓶安全距离计算、火焰与安全帽及人脸授权、低照度增强与告警联动等关键功能模块的设计与实现，并给出主要公式与流程，形成“视频采集—智能分析—决策—告警”的闭环监测体系。

---

## 3.1 多目标检测与气瓶安全距离计算

### 3.1.1 统一检测模型设计

为实现气瓶、火焰/火星、安全帽及人员等多类目标的协同感知，本文采用单阶段目标检测模型 YOLOv8/YOLOv11（Ultralytics）构建统一检测器。模型在一次前向传播中同时输出氧气瓶、乙炔瓶、火焰、火星、安全帽、未佩戴安全帽的人员等类别的检测结果，在保证实时性的前提下便于与后续气瓶测距、火焰多帧判定、安全帽佩戴判定及人脸裁剪等模块对接。

检测结果采用边界框、类别与置信度的统一表示。设第 \(i\) 个检测目标输出为 \((b_i, c_i, p_i)\)，其中 \(b_i = (x_1^{(i)}, y_1^{(i)}, x_2^{(i)}, y_2^{(i)})\) 为边界框在图像坐标系中的左上角与右下角像素坐标，\(c_i\) 为类别标识，\(p_i \in [0,1]\) 为置信度。网络结构包含 Backbone（特征提取）、Neck（多尺度特征融合）与 Head（多尺度检测头），如【图3-1】所示。

【图3-1】YOLO 统一检测模型结构示意图。建议内容：自左向右或自上而下画出 Backbone、Neck、Head 三部分；标出输入（图像）、多尺度特征、检测头输出（边界框、类别、置信度）。】

边界框回归采用 Complete IoU（CIoU）损失，在重叠面积基础上考虑中心点距离与宽高比一致性，便于提升定位精度。设预测框与真实框的 IoU 为 \(\rho\)，中心点欧氏距离为 \(d\)，最小外接矩形对角线长度为 \(c\)，则 CIoU 损失可写为式 (3-1)：

\[
\mathcal{L}_{\text{CIoU}} = 1 - \rho + \frac{d^2}{c^2} + \alpha v
\tag{3-1}
\]

其中 \(v\) 为宽高比一致性项，\(\alpha\) 为权重系数。分类损失采用交叉熵。训练数据来源于公开数据集与自采标注数据，按 YOLO 格式组织；数据增强包括尺度、色彩与裁剪等，以提升在矿山场景下的泛化能力。推理时设置置信度阈值与 NMS 阈值，过滤低置信度与重叠框，输出最终检测列表供下游模块使用。

### 3.1.2 单目测距与安全间距判定

气瓶安全距离判定依赖单目视觉下的空间位置估计。本文采用针孔相机模型，通过相机标定获得内参矩阵（焦距 \(f_x, f_y\)、光心 \(c_x, c_y\)）与畸变系数。像素坐标与世界坐标的几何关系如【图3-2】所示。

【图3-2】针孔模型与单目测距几何关系。建议内容：示意图中含光心、成像平面、物体；标出物高 \(H_{\text{real}}\)、像高 \(h_{\text{pixel}}\)、焦距 \(f\)、纵向距离 \(D\)；可标出相似三角形关系。】

在已知气瓶实际高度 \(H_{\text{real}}\)（如氧气瓶约 1.45 m、乙炔瓶约 1.05 m）的条件下，由检测框在图像中的像素高度 \(h_{\text{pixel}}\) 与相似三角形关系，可得相机到该气瓶的纵向距离（深度）为式 (3-2)：

\[
D = \frac{f \cdot H_{\text{real}}}{h_{\text{pixel}}}
\tag{3-2}
\]

其中 \(f\) 取 \(f_y\)（垂直方向焦距），\(h_{\text{pixel}}\) 为边界框的像素高度 \((y_2 - y_1)\)。将边界框中心或底边中心结合深度投影到相机坐标系，可得三维点 \((X, Y, Z)\)。设像素坐标为 \((u, v)\)，深度为 \(Z\)，则：

\[
X = \frac{(u - c_x) \cdot Z}{f_x}, \quad Y = \frac{(v - c_y) \cdot Z}{f_y}
\tag{3-3}
\]

两气瓶在空间中的欧氏距离为式 (3-4)：

\[
d_{12} = \sqrt{(X_1 - X_2)^2 + (Y_1 - Y_2)^2 + (Z_1 - Z_2)^2}
\tag{3-4}
\]

为抑制检测抖动带来的距离波动，对连续多帧的距离估计进行时间滤波（如滑动平均），以滤波后的距离作为当前时刻的安全间距判定依据。依据 GB 30871—2022，当任意一对氧/乙炔瓶间距小于 5 m 时，生成“气瓶安全距离违规”类安全事件并触发告警。

## 3.2 火焰、安全帽与人脸授权模块

### 3.2.1 火焰与多帧一致性判定

火焰与火星在统一检测模型中作为独立类别输出。为避免矿灯反射、焊接弧光等造成的单帧误报，本文引入多帧一致性判定：在设定时间窗口内，当连续 \(N\) 帧均在对应区域检测到火焰/火星且置信度高于阈值 \(\tau\) 时，才确认火焰事件并上报告警；否则将计数器清零，不触发告警。判定逻辑可形式化为式 (3-5)：

\[
\text{告警触发} \Leftrightarrow \exists t,\; \forall k \in \{0,1,\ldots,N-1\},\; p_{\text{fire}}(t-k) \geq \tau
\tag{3-5}
\]

其中 \(p_{\text{fire}}(t)\) 表示第 \(t\) 帧火焰检测的置信度（或区域最大置信度）。本文取 \(N=3\)，\(\tau\) 可根据现场情况配置。烟雾采用相同多帧逻辑，以较低严重等级（WARNING）纳入规则引擎。多帧一致性判定流程如【图3-3】所示。

### 3.2.2 安全帽佩戴判定

安全帽佩戴状态通过“人员”与“安全帽”检测框的空间关联判定。统一检测模型同时输出人员（或未佩戴安全帽人员）与安全帽的边界框。对每个人员框，根据人体比例近似确定头部区域：取框内上部一定比例区域（如高度比 \(\eta \in (0,1)\)）作为头部候选区。若某安全帽框的中心点 \((x_c, y_c)\) 落在该人员框的头部区域内，则判定该人员已佩戴安全帽，否则判定未佩戴。形式化地，设人员框为 \([x_1^p, y_1^p, x_2^p, y_2^p]\)，头部区域下边界为 \(y_h = y_1^p + \eta (y_2^p - y_1^p)\)，则佩戴判定为式 (3-6)：

\[
\text{已佩戴} \Leftrightarrow x_1^p \leq x_c \leq x_2^p \,\wedge\, y_1^p \leq y_c \leq y_h
\tag{3-6}
\]

未佩戴时生成“人员防护违规”类安全事件并触发告警。该方法无需额外姿态估计模型，计算量小，便于与现有检测管线集成。安全帽与人员框的空间关联示意如【图3-4】所示。

【图3-4】安全帽佩戴判定的空间关联示意。建议内容：示意图中含人员边界框、头部区域（框内上部阴影）、安全帽框及其中心点；标出“中心点落入头部区域→已佩戴”“未落入→未佩戴”。】

### 3.2.3 人脸授权与闯入检测

人脸授权模块实现作业人员身份核验与非授权闯入检测。对检测到的人员，在其边界框内进行人脸检测与裁剪；对裁剪后的人脸图像，使用深度人脸特征模型（如 DeepFace 封装的 ArcFace）提取归一化特征向量 \(\mathbf{f} \in \mathbb{R}^d\)。与预存授权库中第 \(j\) 个身份的特征 \(\mathbf{f}_j\) 比对时，采用余弦相似度式 (3-7)：

\[
s_{ij} = \frac{\mathbf{f} \cdot \mathbf{f}_j}{\|\mathbf{f}\| \|\mathbf{f}_j\|}
\tag{3-7}
\]

若 \(\max_j s_{ij} \geq \theta\) 且满足授权策略，则判定为授权人员；否则判定为未授权或陌生人，生成“非授权闯入”类安全事件并触发告警。ArcFace 在训练阶段通过加性角度间隔损失增强特征判别性，推理时上述余弦相似度即可用于比对。授权库支持通过管理端或离线方式录入授权人员人脸图片并提取特征存储。井下低照度条件下，可先对 ROI 进行增强再做人脸检测与识别。人脸授权流程如【图3-5】所示。

## 3.3 低照度增强与告警联动

### 3.3.1 低照度图像增强

在井下或夜间等低照度场景下，原始图像对比度低、噪声明显，会降低检测与识别的稳定性。本文在预处理阶段提供可选的低照度增强模块，采用基于直方图均衡的方法对亮度通道进行增强，在保证实时性的前提下提升可见度与对比度。

**直方图均衡原理**  
设输入灰度图像共有 \(L\) 个灰度级（如 \(L=256\)），灰度级 \(r_k\)（\(k=0,1,\ldots,L-1\)）的像素个数为 \(n_k\)，图像总像素数为 \(n\)，则 \(r_k\) 出现的概率为 \(p_r(r_k) = n_k / n\)。直方图均衡将灰度 \(r_k\) 映射为 \(s_k\)，使输出直方图趋于均匀，其变换为累积分布函数（CDF）的线性缩放，即式 (3-8)：

\[
s_k = T(r_k) = (L-1) \sum_{i=0}^{k} p_r(r_i) = (L-1) \cdot \text{CDF}(r_k)
\tag{3-8}
\]

其中 \(\text{CDF}(r_k) = \sum_{i=0}^{k} p_r(r_i)\) 为输入灰度的累积分布。式 (3-8) 将输入动态范围映射到 \([0, L-1]\)，使输出灰度近似服从均匀分布，从而拉伸对比度。全局直方图均衡（HE）直接按整幅图像统计直方图并应用式 (3-8)，对低照度整体偏暗的图像有提亮效果，但易放大噪声且对局部光照不均适应性差。

**CLAHE 与实现**  
限制对比度自适应直方图均衡（CLAHE）在局部区域（如 \(8\times8\) 的网格块）内分别做直方图均衡，并对每个块的直方图进行截断限制（clip limit）：将超出阈值的直方图 bin 截断后重新分配，再按式 (3-8) 的形式计算该块内的映射 \(s = T_{\text{loc}}(r)\)，从而在增强对比度的同时抑制噪声过度放大。像素点的最终灰度由相邻块映射结果双线性插值得到。本文实现中，将输入帧由 BGR 转至 LAB 色彩空间，仅对明度通道 **L** 应用 CLAHE（或可选 HE），保持色度通道 A、B 不变以减小色偏；增强后的 L 与 A、B 合并后转回 BGR，再送入统一检测模型与人脸识别模块。CLAHE 参数包括截断阈值（clip limit，如 2.0）与网格尺寸（tile grid size，如 \(8\times8\)），可根据现场光照与实时性要求配置；是否启用增强也可由用户开关控制。低照度增强在检测管线中的位置如【图3-6】所示。

### 3.3.2 事件抽象与告警联动

各检测与判定模块（气瓶距离、火焰、安全帽、人脸授权）的输出统一抽象为“安全事件”数据结构，包含事件类型、严重等级、时间戳、设备/摄像头标识、可选描述信息及元数据（如置信度、边界框、距离值等）。规则引擎根据事件类型与等级决定是否触发本地声光告警、是否上传至云端以及是否在管理端高亮展示。边缘端将告警事件及关联的证据图片通过 REST API 提交至后端，后端落库并推送给已连接的管理端；管理端支持告警列表查看、筛选、确认及统计，形成“监测—分析—决策—告警”的闭环。事件与告警联动关系如【图3-7】所示。

【图3-7】安全事件与告警联动关系。建议内容：左侧为事件类型（气瓶距离违规、火焰、烟雾、安全帽未佩戴、非授权闯入），右侧为联动动作（本地声光、上传云端、管理端展示）；中间为规则引擎与等级映射。】

系统采用模块化与事件驱动设计，便于后续增加新事件类型或调整告警策略而无需改动核心检测管线。
