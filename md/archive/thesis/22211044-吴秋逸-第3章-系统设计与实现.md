# 3 系统设计与实现

本章在需求分析的基础上给出系统总体架构与数据流设计，并分别阐述多目标检测与气瓶安全距离计算、火焰与安全帽及人脸授权模块、低照度增强与告警联动等关键模块的设计与实现，形成“视频采集—智能分析—决策—告警”的闭环监测体系。

---

## 3.1 需求分析与总体架构

### 3.1.1 功能需求

结合 GB 30871—2022《危险化学品企业特殊作业安全规范》[1]及矿山动火作业实际流程，系统需实现以下核心功能：（1）气瓶安全距离监测：对氧气瓶、乙炔瓶进行检测与定位，基于单目视觉估计气瓶间欧氏距离，当间距小于规范要求的 5 m 时触发告警。（2）火焰/火星监测：对明火与飞溅火星进行检测，并通过多帧一致性判定降低矿灯反射、焊接弧光等造成的误报。（3）安全帽佩戴监测：识别画面中的作业人员与安全帽，通过人员框与安全帽框的空间关联判定佩戴状态，未佩戴时触发告警。（4）人员身份授权：对进入作业区域的人员进行人脸识别，与授权库比对，非授权人员闯入时触发告警。（5）低照度适应性：在井下等低照度场景下，通过图像增强保障上述检测的稳定性。（6）告警联动与管理端展示：将各模块输出统一抽象为安全事件，经规则引擎触发声光告警，并在 Web 管理端进行实时展示与历史查询。

非功能需求方面，系统需具备实时性（视频帧处理延迟可接受）、可部署性（边缘端可独立运行、与云端协同），以及模块化与可扩展性，便于后续增加检测类别或告警规则。

### 3.1.2 系统架构与数据流

系统采用“边缘—云端”分层架构。边缘侧为 AI 边缘节点（Edge Node），负责视频拉流（本地摄像头或 RTSP 等）、图像预处理、多目标检测、气瓶测距、火焰/安全帽/人脸等逻辑判定及本地声光告警，并将告警事件与证据图片上传至云端。云端由后端服务与 Web 管理端组成：后端提供设备管理、告警接收与存储、用户与任务管理、视频流协调等 REST API；管理端为 Vue 3 单页应用，供操作员与管理员查看设备状态、告警列表与统计、任务及实时视频流。边缘与云端通过 HTTP/HTTPS 通信，边缘使用设备令牌（X-Device-Token）认证，管理端使用 JWT 认证。

数据流可概括为：视频输入 → 预处理（含可选低照度增强）→ 统一目标检测（气瓶、火焰/火星、安全帽、人员及人脸区域）→ 安全规则引擎（气瓶距离、火焰多帧判定、安全帽空间关联、人脸授权比对）→ 安全事件生成 → 本地告警与云端上报 → 管理端展示与确认。各模块输出统一抽象为带类型、严重等级、时间戳及可选元数据（如置信度、边界框）的安全事件，便于规则引擎与上层应用一致处理。

## 3.2 多目标检测与气瓶安全距离计算

### 3.2.1 统一检测模型

为实现气瓶、火焰/火星、安全帽及人员等多类目标的协同感知，本文采用单阶段目标检测模型 YOLOv8/YOLOv11（Ultralytics）构建统一检测器。模型在一次前向传播中同时输出氧气瓶、乙炔瓶、火焰、火星、安全帽、未佩戴安全帽的人员等类别的边界框与置信度，在保证实时性的前提下便于与后续气瓶测距、火焰多帧判定、安全帽佩戴判定及人脸裁剪等模块对接。网络结构包含 Backbone（特征提取）、Neck（多尺度特征融合）与 Head（多尺度检测头），损失函数采用边界框回归（如 CIoU）与分类交叉熵的组合。训练数据来源于公开数据集与自采标注数据，按 YOLO 格式组织；数据增强包括尺度、色彩与裁剪等，以提升在矿山场景下的泛化能力。推理时设置置信度与 NMS 阈值，过滤低置信度与重叠框，输出最终检测列表供下游模块使用。

### 3.2.2 单目测距与安全间距判定

气瓶安全距离判定依赖单目视觉下的空间位置估计。本文采用针孔相机模型，通过相机标定获得内参矩阵与畸变系数；在已知气瓶实际高度（或采用典型值）的条件下，利用检测框在图像中的像素高度与相似三角形关系，估计相机到各气瓶的纵向距离。将检测框底部中心或底面投影到假设的地面平面，结合相机外参或安装高度与俯仰角，将像素坐标转换到统一世界坐标系，计算两气瓶间的欧氏距离。为抑制检测抖动带来的距离波动，对连续多帧的距离估计进行时间滤波（如滑动平均），以滤波后的距离作为当前时刻的安全间距判定依据。依据 GB 30871—2022[1]，当任意一对氧/乙炔瓶间距小于 5 m 时，生成“气瓶安全距离违规”类安全事件并触发告警。单目测距的实现参考了基于检测与测距一体化的思路[11]，在工程上通过标定与滤波平衡精度与实时性。

## 3.3 火焰、安全帽与人脸授权模块

### 3.3.1 火焰与多帧一致性判定

火焰与火星在统一检测模型中作为独立类别输出。为避免矿灯反射、焊接弧光等造成的单帧误报，本文引入多帧一致性判定：在设定时间窗口内，当连续多帧均在对应区域检测到火焰/火星且置信度高于阈值时，才确认火焰事件并上报告警；否则视为噪声或瞬时干扰，不触发告警。时间窗口长度与所需连续帧数可根据现场情况配置，在降低误报的同时兼顾对真实火情的响应速度。烟雾可作为较低严重等级的告警类别一并纳入规则引擎。

### 3.3.2 安全帽佩戴判定

安全帽佩戴状态通过“人员”与“安全帽”检测框的空间关联判定。统一检测模型同时输出人员（或未佩戴安全帽人员）与安全帽的边界框。对每个人员框，根据人体比例近似确定头部区域（如框内上部一定比例区域）；若某安全帽框的中心点落在该人员框的头部区域内，则判定该人员已佩戴安全帽，否则判定未佩戴。未佩戴时生成“人员防护违规”类安全事件并触发告警。该方法无需额外姿态估计模型，计算量小，便于与现有检测管线集成，与文献中基于空间关联的佩戴判定思路一致[16,19]。

### 3.3.3 人脸授权与闯入检测

人脸授权模块实现作业人员身份核验与非授权闯入检测。对检测到的人员，在其边界框内进行人脸检测与裁剪；对裁剪后的人脸图像，使用深度人脸特征模型（如 DeepFace 所封装的 ArcFace[21]）提取归一化特征向量，与预存的授权人员特征库进行相似度比对（如余弦相似度）。若相似度高于设定阈值且满足授权时间与区域等策略，则判定为授权人员；否则判定为未授权或陌生人，生成“非授权闯入”类安全事件并触发告警。授权库支持通过管理端或离线方式录入授权人员人脸图片并提取特征存储。井下低照度条件下，可先对 ROI 进行增强再做人脸检测与识别，以提升检出率与识别率[20]。

## 3.4 低照度增强与告警联动

### 3.4.1 低照度图像增强

在井下或夜间等低照度场景下，原始图像对比度低、噪声明显，会降低检测与识别的稳定性。本文在预处理阶段提供可选的低照度增强模块：优先采用传统或轻量方法（如 CLAHE、Gamma 校正）对输入帧进行增强，在保证实时性的前提下提升可见度与对比度；若算力允许，可集成轻量级深度增强模型（如 Zero-DCE 类方法[23]）作为补充。增强后的图像再送入统一检测模型与人脸识别模块。增强强度与是否启用可根据现场光照情况配置，避免过增强引入伪影或影响检测精度。

### 3.4.2 事件抽象与告警联动

各检测与判定模块（气瓶距离、火焰、安全帽、人脸授权）的输出统一抽象为“安全事件”数据结构，包含事件类型、严重等级、时间戳、设备/摄像头标识、可选描述信息及元数据（如置信度、边界框、距离值等）。规则引擎根据事件类型与等级决定是否触发本地声光告警、是否上传至云端以及是否在管理端高亮展示。边缘端将告警事件及关联的证据图片通过 REST API 提交至后端，后端落库并推送给已连接的管理端；管理端支持告警列表查看、筛选、确认及统计，形成“监测—分析—决策—告警”的闭环。系统采用模块化与事件驱动设计，便于后续增加新事件类型或调整告警策略而无需改动核心检测管线。
