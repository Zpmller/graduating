# 4 实验与结果分析

本章从实验环境与数据集、检测与测距性能、系统功能与告警测试三个方面对系统进行验证与分析，评估所建系统是否满足矿业动火作业安全监测的实际需求。

---

## 4.1 实验环境与数据集

### 4.1.1 硬件与软件环境

实验所用硬件与软件环境如表 4-1 所示。边缘侧采用配备 NVIDIA GPU 的工控机或台式机，用于运行视频拉流、YOLO 检测、人脸识别及告警逻辑；云端后端部署于服务器，运行 FastAPI 与 MySQL；管理端在普通 PC 浏览器中访问。软件方面，边缘端采用 Python 3.8+，使用 PyTorch、Ultralytics YOLOv8/YOLOv11、OpenCV、DeepFace 等库；后端采用 FastAPI、SQLAlchemy、Alembic；前端采用 Vue 3、Vite、TypeScript、Element Plus。视频输入为本地摄像头或 RTSP 流，分辨率与帧率可根据现场配置（如 640×480 或 1280×720，15–30 fps）。

### 4.1.2 数据集构成

多目标检测模型的训练与验证数据按 YOLO 格式组织，由多类数据源合并构成，均存放于边缘端 `ai_edge_system/data` 目录下。具体所用数据集如表 4-2 所示。

**表 4-2 统一检测模型所用数据集（ai_edge_system/data 下）**

| 子目录 | 数据集名称 | 对应检测类别 | 说明 |
|--------|------------|--------------|------|
| face | Face detection with yolov8.v2i.yolov11 | face | 人脸检测，用于人脸区域裁剪以支撑授权模块 |
| fire | Fire and smoke detection.v2i.yolov11 | fire, smoke | 火焰与烟雾检测，动火场景明火与烟雾 |
| fire | spark.v2i.yolov11 | sparks | 火星检测，动火飞溅火星小目标 |
| gas_cylinder | CylinDeRS | gas_cylinder | 工业气瓶数据集，氧气瓶、乙炔瓶等 |
| helmet | Construction | helmet, person, no-helmet | 安全帽与施工场景，安全帽、人员、未佩戴安全帽人员 |

训练时按 `data.yaml` 将上述数据集的 train/valid/test 路径合并为统一训练集、验证集与测试集；spark 仅参与训练，验证与测试沿用 Fire and smoke detection 等划分。统一模型共 8 类：face、fire、smoke、sparks、gas_cylinder、helmet、person、no-helmet。数据做尺寸归一化与增强后用于训练与评估。人脸识别模块的授权库由授权人员的人脸图片经 DeepFace/ArcFace 提取特征后离线构建；测试时使用与检测测试集无交集的现场或模拟现场人脸图像。相机标定采用棋盘格标定法，获取用于单目测距的内参与畸变系数。

## 4.2 检测与测距性能

### 4.2.1 多目标检测效果

在测试集上对统一检测模型进行评估，采用 mAP（mean Average Precision）、精确率、召回率以及推理速度（FPS）等指标。本次实验采用 YOLOv11 在表 4-2 所列数据集上训练 100 个 epoch（run：mining_safety_v1），验证集整体指标如表 4-3 所示；检测性能曲线采用该 run 下生成的曲线图，如图 4-1、图 4-2 所示。

**表 4-3 统一检测模型验证集性能（mining_safety_v1，epoch 100）**

| 指标 | 数值 | 说明 |
|------|------|------|
| mAP50 | 0.794 | IoU=0.5 时的平均精度 |
| mAP50-95 | 0.598 | IoU 0.5～0.95 平均 |
| Precision | 0.845 | 精确率 |
| Recall | 0.750 | 召回率 |

实验表明，在气瓶、火焰/火星、安全帽及人员等类别上，模型能够达到可接受的检测精度与实时性，满足现场视频流在线推理需求。部分类别（如小目标火星、远距离气瓶）在困难样本上存在漏检或误检，通过调整置信度阈值、增强困难样本训练或引入多尺度检测可进一步改善。与仅针对单一类别训练的基线相比，统一多类别检测在保持单类性能的前提下减少了模型数量与推理次数，有利于边缘部署。

### 4.2.3 火焰多帧判定与安全帽、人脸模块

火焰多帧一致性判定（式 (3-5)，\(N=3\)）有效降低了单帧误报（如弧光、反光）导致的虚假告警，在测试视频中真实火情能被及时识别并告警。安全帽佩戴判定模块在人员与安全帽均被正确检测的前提下，空间关联规则（式 (3-6)）能够稳定区分佩戴与未佩戴状态；在遮挡或极端角度下偶有误判，可通过优化检测或引入启发式规则改进。人脸授权模块在光照条件较好的测试集上识别率与拒识率可满足现场试用要求；在低照度或侧脸情况下，结合 ROI 增强与多帧融合可提升稳定性[20,24]。

**（1）系统启动顺序**  
建议先启动后端服务（FastAPI + MySQL），再启动边缘端程序，最后在浏览器中打开管理端并登录。边缘端启动后若已配置后端地址与设备令牌，会自动拉取设备配置并在日志区显示“已加载设备配置”，视频源输入框可能被自动填入后端下发的地址（见图 4-3 右侧日志区）。

**（2）边缘端使用说明**  
边缘端主界面左侧为视频预览，右侧为配置与日志（见图 4-3）。**视频源**：在 Video Source 分组中输入摄像头索引（如 0）、RTSP/HTTP 地址或本地视频路径，也可点击 Browse Video File 选择文件。**开始监测**：点击 Start Monitoring，画面将显示带检测框与类别标签的实时结果，发生告警时界面红框提示并蜂鸣，告警同时上报至后端。**配置参数**：Configuration Parameters 中可调整置信度阈值（Confidence）、气瓶最小安全距离（Min Cyl Dist，默认 5 m）、多帧一致性帧数（Consistency，默认 3）、告警冷却时间（Alert Cooldown）。**用户注册**：User Registration 中填写姓名后，通过 Register from Camera 或 Register from File 将授权人员人脸加入本地库，注册后可上传至后端；点击“从后端同步人脸库”可将后端已维护的人脸库同步到本机。**相机标定**：Tools 中的 Camera Calibration 用于单目测距标定，标定后可更准确估计气瓶间距。**日志**：System Logs & Alarms 区域显示设备配置、人脸同步、告警上报等日志，便于排查。

**（3）管理端使用说明**  
管理端为 Vue 3 单页应用，侧边栏包含监控面板、历史记录、设备管理、任务管理、用户管理（管理员）。**登录**：打开管理端地址，输入账号密码登录后进入监控面板（对应图 4-4、4-5 的入口）。**设备管理**（见图 4-4）：展示已注册边缘设备的名称、位置、IP、在线状态、最后心跳时间；管理员可新增、编辑、删除设备，创建设备后需将生成的 Device Token 配置到对应边缘端。**历史记录**（见图 4-5）：展示上报的告警列表，支持按设备、类型、等级、时间筛选与分页；每条告警含类型、等级、设备、时间、证据图及确认状态，操作员可点击确认或查看大图。**实时视频**（见图 4-6，若已配置流媒体）：在设备列表或详情中可查看该设备实时视频流及检测叠加。**任务管理**（见图 4-7）：点击侧边栏“任务管理”进入任务页。页面展示任务列表（卡片或表格），每项包含标题、描述、优先级（高/中/低）、状态（待处理/进行中/已完成）、负责人、截止日期、创建时间；顶部提供“创建任务”“刷新”。点击“创建任务”或某任务的“编辑”可在弹窗中填写或修改标题、描述、优先级、状态、分配人、截止日期；可对任务进行“删除”。用于维护现场巡检、设备维保、告警跟进等任务，便于与告警联动与责任到人。**用户管理**（见图 4-8）：仅管理员角色可见。页面以表格展示所有可登录管理端的用户，列包括用户名、姓名、角色（管理员/操作员/查看者）、状态（激活/禁用）；顶部提供“添加用户”“刷新”。点击“添加用户”或某用户的“编辑”可在弹窗中填写或修改用户名、姓名、角色、密码（新建必填）；可对用户“激活/禁用”或“删除”（不能删除当前登录账号）。角色决定权限：管理员可管理设备与用户，操作员可确认告警与查看设备，查看者仅可查看。

**（4）典型使用流程**  
管理员在管理端创建设备并记录 Device Token，必要时在“用户管理”中创建操作员账号、在“任务管理”中创建巡检或维保任务 → 在边缘端配置后端地址与 Token 后启动，确认日志中已加载设备配置 → 设置视频源并点击 Start Monitoring → 在 User Registration 中录入授权人员人脸或从后端同步人脸库；操作员在管理端“历史记录”中查看与确认告警，在“设备管理”中查看设备状态，在“任务管理”中处理分配给自己的任务。需要更精确的气瓶距离时，使用边缘端 Camera Calibration 完成标定，并在 Configuration Parameters 中微调参数。
