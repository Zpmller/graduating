### 各章内容说明

- **第 1 章 引言**：1.1 写研究背景与意义；1.2 写国内外研究现状，按 1.2.1 / 1.2.2 分方向综述，1.2.3 小结并指出不足；1.3 简要交代全文各章安排（文章组织结构）。
- **第 2 章 面向矿业动火作业的安全监测系统设计**：偏“设计”，含设计目标、总体设计（功能结构 + 逻辑结构）、边缘子系统与云端子系统设计、规则与告警联动、子系统间接口，章末加 2.7 本章小结。
- **第 3 章 关键功能模块设计与实现**：偏“实现”，写多目标检测与气瓶距离、火焰/安全帽/人脸授权、低照度与告警联动等关键机制的设计与实现，章末加 3.4 本章小结。
- **第 4 章 实验与结果分析**：实验环境与数据集、检测与测距性能、系统功能与告警测试，章末加 4.4 本章小结。
- **第 5 章 结论**：总结全文工作、主要结论、不足与展望，不引入新内容。

**与现有各章 md 的对应**：现有「引言」可拆为 1.1 / 1.2 / 1.3；「第2章 相关技术与研究现状」对应 1.2 的展开版，或保留为第 2 章前的技术综述并按新结构并入 1.2；若学校要求“研究现状”独立成章，可将 1.2 改为第 2 章、原“系统设计”顺延为第 3 章。现有「第3章 系统设计与实现」拆为：总体设计、子系统设计、接口设计 → 新第 2 章；关键模块实现 → 新第 3 章。现有「第4章 实验与结果分析」「第5章 结论」可直接对应新第 4、5 章，仅补各章小结即可。

## 二、公式与研究方案放在哪里

### 2.1 公式（公式）放哪里

公式随**所讲原理或算法**放在对应小节，**集中在第 3 章**，必要时在第 2 章总体设计中给出关键关系式（不展开推导）。

| 内容 | 建议放置位置 |
|------|----------------|
| 目标检测（边界框、类别、置信度表示） | **3.1.1 统一检测模型设计** |
| 边界框回归、CIoU 损失 | **3.1.1 统一检测模型设计** |
| 针孔模型、像素与空间坐标关系 | **3.1.2 单目测距与安全间距判定** |
| 相似三角形测距公式 \(D = f \cdot H_{real} / H_{pixel}\)、气瓶间欧氏距离、时间滤波 | **3.1.2 单目测距与安全间距判定** |
| 多帧一致性判定（连续 N 帧置信度超阈值） | **3.2.1 火焰与多帧一致性判定** |
| 安全帽佩戴（头部区域、中心点落入判定） | **3.2.2 安全帽佩戴判定** |
| 人脸特征嵌入、ArcFace 损失或余弦相似度判定 | **3.2.3 人脸授权与闯入检测** |
| 低照度增强（Gamma、CLAHE、Retinex 等） | **3.3 低照度增强与告警联动** |
| 事件抽象、规则触发条件（若需形式化） | **2.5.1 规则引擎设计** 或 **3.3** |

**说明**：第 1 章、第 2 章以文字和框图为主，不堆公式；第 3 章在讲清“怎么做”的同时给出上述公式并编号（如 (3-1)、(3-2)），便于正文与实验引用。

---

### 2.2 研究方案（开题里的内容）放哪里

开题报告中的「研究方案」通常包含：**理论基础**、**研究方法**、**研究步骤**、**预期成果**。在正文中的对应关系如下：

| 开题报告部分 | 放入正文位置 | 说明 |
|--------------|--------------|------|
| **理论基础**（目标检测、针孔模型、CIoU、ArcFace、多帧判定、安全帽关联、低照度模型等） | **第 3 章** 各对应小节 | 与“设计与实现”一起写：先给原理与公式，再写本系统如何用。不单独设“理论基础”章，避免与第 1 章重复。 |
| **研究方法**（为何选 YOLO、单目测距、多帧判定、空间关联、DeepFace/ArcFace、CLAHE 等） | **第 2 章 2.1 系统设计目标**、**2.2 系统总体设计**；部分在 **第 3 章** 各节开头 | 2.1 / 2.2 写设计目标、技术路线与总体方法选型；第 3 章各节开头可简要写“本模块采用……方法”，再展开公式与实现。 |
| **研究步骤**（需求分析→模型构建→系统实现→实验验证） | **第 2 章 2.2 系统总体设计**（技术路线/流程）；**第 3、4 章** 的章节顺序即步骤展开 | 2.2 用一段或框图写“需求分析—总体设计—关键模块实现—实验验证”；第 3 章对应“模型构建+系统实现”，第 4 章对应“实验验证”。 |
| **预期成果** | **第 1 章 1.1 或 1.3** 末尾简要提及；**第 5 章 结论** 对照实际写 | 引言中不展开；结论中写“达到了……”“实现了……”，与开题预期呼应即可。 |

**小结**：**公式**随原理放在**第 3 章**对应小节；**研究方案**中的理论基础与公式进**第 3 章**，研究方法与步骤进**第 2 章**（2.1 / 2.2），预期成果在**第 1 章**点一句、在**第 5 章**总结。

## 三、引言（第 1 章）写什么

引言应**简要说明**：研究目的与范围、相关领域前人工作与知识空白、理论基础与分析、研究设想与实验设计、预期结果与意义。**言简意赅，不与摘要雷同，不做摘要的注释；教科书常识不必展开。**

建议按下面几块组织（每块一两段即可）：

### 1. 研究目的与范围

- **目的**：矿业动火作业（气割、焊接等）风险高，规范执行长期依赖人工巡检与现场监护，难以对气瓶间距、火源、人员防护与准入等做**持续、客观的量化监测**；本文旨在构建一套基于 AI 视觉的监测系统，实现对上述多类风险的协同感知与实时预警。
- **范围**：面向矿山（含井下）动火作业场景，以 GB 30871—2022 等现行规范为约束，覆盖气瓶安全距离、火焰/火星、安全帽佩戴、作业人员身份授权四类安全目标，以及低照度下的检测稳定性与告警联动；不涉及非视觉传感器或非动火类作业的全面安全评估。

### 2. 相关领域前人工作与知识空白

- **前人工作**：国内外在火焰检测、安全帽检测、气瓶检测、人脸识别及低照度增强等方面已有大量研究；YOLO 等目标检测与单目测距在工业场景中逐步应用；视觉手段用于矿山/化工安全监管已有探索。
- **知识空白**：针对**矿业动火作业**这一多风险耦合场景，将气瓶安全距离、火焰、防护与人脸授权纳入**统一视觉框架**、并将识别结果**直接映射到现行安全规范量化指标**（如氧乙炔瓶间距≥5 m）的系统化工作仍不足；规范条款的自动判定与连续监测能力欠缺，制约“落实到位”。

### 3. 理论基础与分析（扼要）

- 不重复教科书内容，只点明本文所依据的几类技术路线：**单阶段目标检测**（YOLOv8/11）统一多类目标；**针孔相机模型**与标定实现单目测距与气瓶间距判定；**多帧一致性**降低火焰误报；**人员框与安全帽框的空间关联**判定佩戴；**深度人脸特征**（如 ArcFace/DeepFace）与授权库比对实现身份与闯入判定；**传统或轻量图像增强**（如 CLAHE）保障低照度下的检测稳定性。

### 4. 研究设想、方法与实验设计

- **设想**：采用“视频采集—智能分析—决策—告警”的闭环；边缘端负责视频与 AI 分析，云端/管理端负责设备管理、告警存储与展示；各模块输出统一抽象为安全事件，经规则引擎触发声光告警与管理端展示。
- **方法**：统一检测模型 + 单目测距 + 多帧火焰判定 + 空间关联安全帽 + 人脸授权 + 低照度增强 + 事件驱动告警；实验上基于自建/公开数据与典型场景进行检测精度、测距误差、误报/漏报及端到端告警测试。

### 5. 预期结果与意义

- **预期结果**：系统能对现场视频进行实时分析，在气瓶距离违规、火焰异常、防护缺失或非授权闯入等情形下自动触发告警，形成“监测—分析—告警”的闭环；各模块可独立运行与扩展。
- **意义**：为矿山动火作业提供可落地的智能监测技术方案，提升安全管控的**客观性**与**实时性**，并为多风险协同感知与规范量化判定提供工程参考。

## 四、与摘要的区分

- **摘要**：概括全文（问题、方法、创新点、结果、关键词），不展开逻辑与文献。
- **引言**：交代**为何做、缺什么、依据什么、怎么做、有何意义**，可简要引用文献指出现状与空白，但不重复摘要的完整方法罗列与创新点表述；细节留给第 2、3、4 章。

按上述结构撰写并控制引言篇幅（通常 1–2 页），即可满足“言简意赅、不与摘要雷同、不为摘要注释”的要求。

## 五、开题报告内容是否放进引言？

**结论：不适合把开题报告整块放进引言，应分流到正文各章。**

| 开题报告部分 | 是否放进引言 | 建议去向 |
|--------------|--------------|----------|
| **文献综述**（气瓶/火焰/安全帽/人脸/低照度等研究现状） | ❌ 不放全文 | 精炼为 1–2 段“前人工作与知识空白”写入引言；**详细综述**放入 **第 2 章 相关技术与研究现状**，按 2.1 / 2.2 / 2.3 展开。 |
| **研究方案·理论基础**（YOLOv8 结构、CIoU、针孔模型、ArcFace 公式等） | ❌ 不放 | 属于正文理论/方法，放入 **第 2 章**（技术原理）或 **第 3 章**（设计与实现）中对应小节，引言只点明“采用单阶段检测、针孔测距、人脸特征比对等”即可。 |
| **研究方案·研究方法 / 研究步骤** | ❌ 不放 | 对应 **第 3 章 系统设计与实现**（需求、架构、各模块方法）和 **第 4 章 实验与结果**（步骤即实验设计）；引言中仅用一两句概括“视频采集—智能分析—决策—告警”闭环及实验思路。 |
| **预期成果** | ⚠️ 可精简放入 | 在引言末尾用 1 段写“预期结果与意义”；全文完成后在 **第 5 章 结论** 中对照实际成果再写，不必与开题完全一致。 |
| **目的、范围、现有不足** | ✅ 适合放入 | 与引言要求的“研究目的、范围、相关领域前人工作与知识空白”直接对应，可基于开题**压缩改写**为 1–2 页，不照抄长段。 |

**原因简述：**

- 引言要求**言简意赅**，篇幅通常 1–2 页；开题报告里的文献综述、理论基础、研究步骤体量很大，全部放进引言会冗长且重复教科书内容。
- 开题是**事前规划**，正文是**事后成文**；引言只需在文首“简要说明”目的、空白、思路与意义，细节和公式应放在第 2、3、4 章展开。
- 把开题内容**分流**到第 2 章（综述+技术）、第 3 章（设计与实现）、第 4 章（实验）后，引言只保留**浓缩版**的目的、范围、空白、设想与意义，既符合规范，又避免与摘要雷同。
