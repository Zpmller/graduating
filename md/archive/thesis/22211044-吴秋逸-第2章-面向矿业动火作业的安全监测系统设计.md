# 2 面向矿业动火作业的安全监测系统设计

本章在遵循 GB 30871—2022 等现行安全规范的前提下，给出面向矿业动火作业的安全监测系统的设计目标、总体架构、边缘与云端子系统的结构与接口、安全规则与告警联动设计，以及子系统间的业务与配置接口，为第 3 章关键功能模块的实现提供设计依据。

---

## 2.1 系统设计目标

系统设计围绕矿业动火作业现场对**持续、客观、可量化**安全监测的需求展开，目标可归纳为功能目标与非功能目标两类。

**功能目标**包括：（1）**气瓶安全距离监测**：对氧气瓶、乙炔瓶进行检测与定位，基于单目视觉估计气瓶间欧氏距离，当间距小于规范要求的 5 m 时触发告警。（2）**火焰/火星监测**：对明火与飞溅火星进行检测，并通过多帧一致性判定降低矿灯反射、焊接弧光等造成的误报。（3）**安全帽佩戴监测**：识别作业人员与安全帽，通过空间关联判定佩戴状态，未佩戴时触发告警。（4）**人员身份授权**：对进入作业区域的人员进行人脸识别并与授权库比对，非授权人员闯入时触发告警。（5）**低照度适应性**：在井下等低照度场景下，通过图像增强保障上述检测的稳定性。（6）**告警联动与管理端展示**：将各模块输出统一抽象为安全事件，经规则引擎触发本地声光告警，并在 Web 管理端进行实时展示与历史查询。

**非功能目标**包括：**实时性**（视频帧处理延迟可接受）、**可部署性**（边缘端可独立运行、与云端协同）、**模块化与可扩展性**（便于后续增加检测类别或告警规则），以及**安全与可维护性**（设备与用户认证、接口清晰、便于运维）。

## 2.2 系统总体设计

### 2.2.1 系统功能结构设计

系统按“边缘—云端”分层，功能划分为边缘侧与云端侧两大块。边缘侧负责视频接入、AI 分析与本地告警；云端侧负责设备管理、告警存储与 Web 看板展示。功能结构如【图2-1】所示。

边缘检测与告警子系统的主要功能为：视频拉流（本地摄像头或 RTSP 等）、图像预处理与可选低照度增强、多目标检测（气瓶、火焰/火星、安全帽、人员及人脸区域）、气瓶安全距离计算、火焰多帧判定、安全帽空间关联判定、人脸授权比对、安全事件生成与规则触发、本地声光告警、告警事件与证据图片上传至云端。云端管理与展示子系统的主要功能为：设备注册与状态管理、告警接收与落库、用户认证与任务管理、视频流协调（与媒体服务器配合）、Web 看板中的设备状态与告警列表展示、告警确认与统计查询。

### 2.2.2 系统逻辑结构设计

系统采用“视频采集—智能分析—决策—告警”的闭环逻辑。数据流为：视频输入 → 预处理（含可选低照度增强）→ 统一目标检测 → 安全规则引擎（气瓶距离、火焰多帧判定、安全帽空间关联、人脸授权比对）→ 安全事件生成 → 本地告警与云端上报 → 管理端展示与确认。系统逻辑架构及数据流如【图2-2】所示。

## 2.3 边缘检测与告警子系统设计

### 2.3.1 系统结构设计

边缘检测与告警子系统部署于现场 AI 边缘节点，采用“表示层—核心逻辑层—业务逻辑层—工具层”的分层结构。表示层负责主界面与配置交互；核心逻辑层包含检测器、人脸识别器、告警上传线程与视频推流模块；业务逻辑层包含安全规则引擎与距离估计器；工具层包含相机标定与低照度增强等。边缘子系统结构如【图2-3】所示。

【图2-3】边缘检测与告警子系统结构图。建议内容：自上而下或分层框图；表示层：主窗口、标定工具；核心逻辑层：检测器(YOLO)、人脸识别器、告警上传器、视频流推送；业务逻辑层：安全规则引擎、距离估计器；工具层：相机内参、图像增强器；标出帧数据、检测结果、规则输入输出、告警事件等数据流向。】

视频处理流程为：采集 → 增强（可选）→ 统一目标检测 → 安全规则引擎（调用距离估计、人脸识别等）→ 安全事件 → 本地告警与上传队列；同时，标注后的帧可经视频流模块推送至媒体服务器供管理端实时查看。

### 2.3.2 系统内部接口设计

边缘子系统内部各模块通过明确的输入输出接口协作。检测器对外提供“帧 → 检测结果列表”的接口；安全规则引擎输入为检测列表与原始帧（用于人脸裁剪），输出为安全事件列表；距离估计器依赖相机内参与检测框，输出气瓶间距离；人脸识别器输入为裁剪的人脸图像，输出为身份与验证结果；告警上传器输入为安全事件与证据图片，通过队列异步上报云端。边缘子系统核心模块的接口定义如表 2-1 所示。

## 2.5 安全规则与告警联动设计

### 2.5.1 规则引擎设计

安全规则引擎位于边缘侧，是“检测结果 → 安全事件”的核心。引擎根据统一检测结果与业务逻辑（气瓶距离、火焰多帧、安全帽关联、人脸授权）判断是否产生安全事件，并赋予事件类型与严重等级。事件类型与后端约定一致，包括：气瓶安全距离违规（distance_violation）、火焰/火星（fire_violation）、烟雾（smoke_violation）、安全帽未佩戴（ppe_violation）、非授权闯入（access_control）。严重等级分为 CRITICAL、DANGER、WARNING，与规范风险和响应优先级对应。

规则引擎采用“事件抽象 + 条件触发”的设计：各检测与判定模块的输出统一抽象为安全事件结构（类型、等级、时间戳、设备标识、描述、元数据）；引擎根据事件类型与等级决定是否触发本地声光告警、是否上传云端、是否在管理端高亮展示。告警联动逻辑如【图2-7】所示。

### 2.6.1 业务接口设计

边缘与云端之间的业务接口以 REST API 为主。边缘节点通过设备令牌（X-Device-Token）调用后端；管理端通过 JWT 调用后端。表 2-4 与表 2-5 分别列出边缘节点调用的接口与管理端调用的接口，便于对接与联调。

**表 2-4 边缘节点调用的云端接口（认证：X-Device-Token）**

| 方法 | 接口路径 | 请求/响应要点 | 说明 |
|------|----------|----------------|------|
| POST | `/api/v1/alerts/` | 请求：multipart/form-data，字段 `alert_data`（JSON 字符串，含 type、level、message、timestamp、metadata）、`image`（证据图片，JPEG/PNG，最大 10MB）；响应：201，返回告警记录（id、device_id、type、severity、timestamp、image_path 等） | 告警上报，触发即发 |

| POST | `/api/v1/devices/{id}/calibration/yaml` | 请求：multipart，字段 `file`（YAML 文件）；响应：200 | 上传本地标定文件至云端 |
| POST | `/api/v1/devices/{id}/calibration/images` | 请求：multipart，字段 `images`（标定用棋盘格图片列表）；响应：200，服务端计算并存储标定 | 上传原始标定图，由云端完成标定 |

### 2.6.2 配置与显示接口设计

配置与显示接口保障管理端对设备与告警的可视化及对运行配置的间接管理，其实现仍基于 2.6.1 中的真实 REST 接口，按用途归纳如表 2-6 所示。

**表 2-6 配置与显示相关接口归纳**

| 用途 | 接口（方法 + 路径） | 说明 |
|------|---------------------|------|
| 设备配置 | PUT `/api/v1/devices/{id}` | 更新设备名称、位置、ip_address 等（Admin） |
| 设备配置 | POST `/api/v1/devices/` | 新建设备并获取 device_token（Admin） |
| 标定配置 | GET/POST `/api/v1/devices/{id}/calibration/yaml` | 边缘拉取/上传标定 YAML |
| 标定配置 | POST `/api/v1/devices/{id}/calibration/images` | 边缘上传标定图片，云端计算标定 |
| 告警展示 | GET `/api/v1/alerts/` | 告警列表、分页与筛选（设备、类型、等级、时间、是否已确认） |
| 告警展示 | GET `/api/v1/alerts/stats` | 按类型/等级/趋势的统计与未确认数量 |
| 告警确认 | PUT `/api/v1/alerts/{id}/acknowledge` | 标记告警已处理 |
| 设备状态展示 | GET `/api/v1/devices/`、GET `/api/v1/devices/{id}` | 设备列表及状态、最后心跳时间 |
| 实时视频 | GET `/api/v1/devices/{id}/stream/offer`，POST `.../stream/answer` | 获取 offer/answer，建立 WebRTC 流 |
| 视频控制 | GET `.../stream/status`，POST `.../stream/control`，DELETE `.../stream` | 查询流状态、切换叠加层/画质、停止流 |

（1）**配置相关**：设备注册与信息更新由管理端经上述设备与标定接口完成；边缘节点可从 GET calibration/yaml 拉取标定或通过本地界面配置阈值、冷却时间等；标定也可由边缘标定工具生成后经 POST calibration/yaml 或 calibration/images 上传。（2）**显示相关**：管理端通过告警列表与 stats 获取告警及聚合数据，通过设备接口获取状态与心跳，通过视频流相关接口获取实时画面并控制叠加层与画质。配置与显示接口不改变 2.6.1 中的认证与业务语义，仅在参数与查询维度上扩展，便于运维与事后分析。

## 2.7 本章小结

本章给出了面向矿业动火作业的安全监测系统的设计目标、总体功能与逻辑结构、边缘检测与告警子系统与云端管理与展示子系统的结构与内部接口、安全规则与告警联动设计，以及边缘与云端之间的业务接口与配置显示接口。系统采用“边缘—云端”分层和“视频采集—智能分析—决策—告警”的闭环设计，各模块通过统一的安全事件抽象与规则引擎衔接，为第 3 章关键功能模块的设计与实现提供了整体框架与接口约束。
