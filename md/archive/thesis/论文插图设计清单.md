## 图1-1 研究技术路线与本文工作定位

**图题**：研究技术路线与本文工作定位  

**插入位置**：第1章「1.3 文章组织结构」前后；若第1章不设图，置于第2章章首。  

**参考风格**：FigureBench **Survey**（二维谱系 + 阶段条）+ **Paper**（顶部横排阶段）；整页可采用「上下分区」或「左右分区」，底色浅灰白，阶段标题条用低饱和蓝/赭色区分（类似参考图中的分栏色带，不必卡通化）。  

**必须出现的内容**

1. **上半或左侧：研究技术路线（5～6 个阶段）**  
   - 阶段框为**等高圆角矩形**，沿一条直线**从左到右**（或从上到下）排列，相邻阶段之间**单一方向粗箭头**连接。  
   - 阶段内文字（可两行内）：  
     ① 问题与规范约束（动火多风险场景；GB 30871—2022 等）；  
     ② 统一视觉感知（YOLO 多类检测；类别与矿业动火一致）；  
     ③ 几何 / 时序 / 身份判定（单目测距；多帧火焰/火星/烟雾；安全帽；人脸授权）；  
     ④ 边缘—云端系统（事件抽象、规则引擎、上报、Web 管理）；  
     ⑤ 实验与系统验证（检测与测距指标；端到端告警；界面与稳定性）。  
   - 每个阶段框**右下角**可加小号章节约束（如「§2」「§3」），字号明显小于框内标题。  

2. **下半或右侧：本文在研究领域中的位置**  
   - 绘制**二维坐标系**：横轴从左到右标注「单任务检测 → 多任务协同感知」；纵轴从下到上标注「算法模块 → 规范映射与可部署系统」。  
   - 坐标平面用**浅淡网格**，象限不必写四象限名称。  
   - 在平面上放置 **3～5 个空心小圆点或缩写标签**，表示「常见工作或研究范式」的示意位置（无需真实文献名，可用「火焰单任务」「PPE 单任务」「通用云边系统」等中性词）。  
   - 用**五角星或圆角高亮框**标出 **「本文」**，落在「多任务协同」与「规范映射与可部署」交汇区域；从星到两轴可画**极短虚线**，方便读坐标。  

3. **图例（独立小框）**  
   - 一项：「阶段箭头：研究推进顺序」；一项：「星标：本文工作位置」。不得使用与数据帧流水线混淆的说明。  

**要素之间关系（须能在图中读出）**

- 阶段序列是**全序**：无回路；若需表达「实验迭代」，仅在阶段⑤旁脚注一行小字，**不画回到阶段② 的粗箭头**。  
- 路线图与二维定位图是**并列信息**：前者回答「做了什么步骤」，后者回答「与别人相比站在哪」；两区之间**留清晰分隔带**（留白或竖线），不要用粗箭头把两区连成因果链。  
- 定位图中「本文」与「其它示意点」是**对比关系**：其它点面积和对比度**弱于**「本文」标记。  

**版式与美观（本图专用）**

- 单图画布建议宽**占版心宽度**，高宽比约 **1:1.3～1:1.5**。  
- 阶段框宽度相等，文字水平居中；箭头线宽一致、弯头曲率一致。  
- 全图**不超过 3 种主色**（如浅蓝条、浅赭条、深灰字），避免荧光色块。

## 图2-1 系统功能结构图

**图题**：系统功能结构图  

**插入位置**：第2章「2.2.1 系统功能结构设计」。  

**参考风格**：FigureBench **Survey** 中「LLM Agent」式 **Perception / 中枢 / 输出** 分区思路（本图改为 **边缘侧 / 云端侧** 两大分区）；模块为圆角框 + 细描边，类似参考架构图中的功能块，不需要插画小人。  

**必须出现的内容**

1. **左右两大分区**（或上下两大分区）  
   - **左：边缘侧** — 浅冷色底（如 #E8EEF5）；**右：云端侧** — 略浅的对比色底（如 #EEF2FF）。  
   - 两区标题条加粗：「边缘检测与告警」「云端管理与展示」。  

2. **边缘侧模块（须全部出现，文字可缩写）**  
   - 视频接入（摄像头 / RTSP / 本地文件）；  
   - 图像预处理（含可选 CLAHE）；  
   - 统一目标检测（YOLO）；  
   - 气瓶距离估计；  
   - 火焰 / 火星 / 烟雾多帧判定；  
   - 安全帽与人员相关判定；  
   - 人脸检测与识别；  
   - 安全规则引擎；  
   - 本地声光告警；  
   - 告警与证据上传；  
   - （可选，虚线框）流媒体推送。  

3. **云端侧模块**  
   - 设备注册与心跳；告警接收与持久化；用户与权限；任务管理；视频流协调（或媒体服务对接）；Web 看板（Vue3）。  

4. **连线与线型**  
   - **实线**：主功能依赖（如：预处理→检测→规则引擎→声光/上传）；  
   - **虚线**：可选能力（流媒体、可选增强）。  
   - **跨分区**：仅 **3～5 条**简明箭头，标注「安全事件/心跳/配置」等，不得画成网状。  

**要素之间关系**

- 边缘侧内部：**检测**向 **规则引擎** 提供结果；**距离估计、人脸识别** 被 **规则引擎** 调用或消费其输出；**上传** 仅接收**已结构化事件与证据**，不从检测直接连到云（除非图注声明特殊调试模式）。  
- 云边关系：**边缘主动上报**，云端**接收存储**；管理端**不直连**边缘模块（视频经协商由另路表示时用脚注）。  

**版式与美观（本图专用）**

- 每侧内模块排成 **2 列 × 多行** 或 **3 列**，对齐网格；**规则引擎**框可比邻居宽 **10%～15%** 作为视觉中心。  
- 模块内文字一行优先，必要时两行；禁止框内段落。

## 图2-2 系统逻辑架构与数据流图

**图题**：系统逻辑架构与数据流图  

**插入位置**：第2章「2.2.2 系统逻辑结构设计」。  

**参考风格**：**Paper** 类参考图中的 **Stage 式横向条带**（ContraNeRF/TDT）：主流程一条水平「粗箭头链」，下方可用浅色条标注数据形态；**不参与**接口路径穷举。  

**必须出现的内容**

1. **主链节点（顺序固定）**  
   视频输入 → 预处理（标注「可选 CLAHE」）→ 统一目标检测 → 安全规则引擎 → 安全事件 → **分叉**：① 本地声光告警；② 云端上报 → 管理端展示与确认。  

2. **从「规则引擎」向下的支链（或向上的输入支链）**  
   - 气瓶间距判定（输入：检测框 + 标定参数；输出：距离值是否违规）；  
   - 火焰/火星/烟雾多帧判定；  
   - 安全帽相关判定；  
   - 人脸授权判定。  
   四条支链在图上**对称分布**（例如上两、下两），汇入同一「规则引擎」框。  

3. **数据标注（贴主链路侧边即可）**  
   原始帧；检测框列表；结构化事件（类型、等级、时间戳、设备、元数据）；证据图像。  

4. **信任边界**  
   用**大虚线框**圈出「边缘」「云端」两大区域；跨云箭头旁标 **X-Device-Token**；进入管理端路径旁标 **JWT**。  

**要素之间关系**

- 主链为 **单向因果**：除脚注说明外不画反馈环。  
- 四条支链与规则引擎：**汇总关系**（引擎根据多路中间结果生成单一事件流）。  
- 「声光」与「上报」对同一事件：**并行扇出**（同一节点引出两条等权箭头）。  

**版式与美观（本图专用）**

- 主链**水平**从左到右，**最多一处折角**；折角角度统一。  
- 「规则引擎」为全图**最深描边**或略大模块。  
- 图宽为主，高适中，避免竖向过长。

## 图2-3 边缘检测与告警子系统结构及内部接口关系

**图题**：边缘检测与告警子系统结构及内部接口关系  

**插入位置**：第2章「2.3」（建议 2.3.1 与 2.3.2 之间）。  

**参考风格**：**Paper** 嵌套框 + **Survey** 分层；左侧或上方写「层次」，同层模块对齐，箭头表达接口（类似 DeepReview 中 Agent 嵌套但简化为四层）。  

**必须出现的内容**

1. **四层泳道（自上而下或自左而右）**  
   - **表示层**：PyQt 主界面、配置与日志；  
   - **核心逻辑层**：检测器、人脸识别器、告警上传线程、视频推流（可选）；  
   - **业务逻辑层**：安全规则引擎、距离估计器；  
   - **工具层**：相机标定、CLAHE 低照度增强。  

2. **数据与控制流箭头**  
   - 视频源 → 预处理 → 检测器 →（检测结果列表）→ 规则引擎；  
   - 规则引擎 ↔ 距离估计器（双向或「请求—返回」双箭，全图统一一种画法）；  
   - 规则引擎 ↔ 人脸识别器；  
   - 规则引擎 → 告警上传 → 队列 → （示意「HTTPS 至云端」）；  
   - 工具层：标定、增强 → 被上层「调用」的箭头指入检测或预处理（与实现一致即可）。  

3. **图注角标**  
   一行：「接口字段详见表2-1」。  

**要素之间关系**

- **层次约束**：上层可依赖下层，下层不反向依赖表示层（调试入口若有，用虚线并标「调试」）。  
- **单一决策出口**：所有可导致告警的路径**必经规则引擎**，再至声光与上传。  

**版式与美观（本图专用）**

- 每层**等高条带**，层间距大于模块间距；跨层线**优先竖线**，减少斜交叉。  
- 同层模块**等宽**，文字垂直居中。
